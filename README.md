# vibe-payment
# Compras e Pagamentos — Protótipo Low Fidelity

## Objetivo
Este projeto tem como objetivo representar, em baixa fidelidade, o fluxo de solicitações de compras e pagamentos avulsos, permitindo leitura, acompanhamento e consolidação de informações antes do processamento no ERP.

## Escopo
- Visualização de solicitações originadas de um formulário público
- Acompanhamento por fases do Kanban
- Registro de comentários livres
- Anexação de evidências
- Indicação de que uma solicitação foi processada no ERP

## Fora de escopo
- Gestão de contratos
- Cálculo de valores variáveis
- Regras financeiras automatizadas
- Integração com ERP
- Automação de fluxo

## Fases do Kanban
- Novas solicitações
- Processando
- Recorrentes
- Arquivados

A fase "Arquivados" indica que a solicitação já foi processada no ERP.

## Origem das solicitações
Todas as solicitações são criadas a partir de um formulário público utilizado pelo comprador.
